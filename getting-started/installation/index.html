    Installation

    Requirements

    • MongoDB server
    • NodeJS
    • Yarn package manager
    • (Optional) A swap file - reason being that Webpack uses loads of memory

    Creating a MongoDB server

    For simplicity, we're going to use mLab with the Free plan. Though it's recommended to [[setup your own MongoDB server|Development server]] if you plan to have a lot of traffic.

    1. Head over to mLab and create an account (make sure to verify your email)
    2. Click on Create new in the upper right corner
    3. Select Sandbox and then click Continue

    4. Select your preferred region and click Continue
    5. Enter injectify as the database name, click Continue and then Submit order
    6. Click on your new database and select Users and then Add database user

    7. Choose a username and password and click Create (make sure read-only is left unchecked)

    Creating a GitHub oauth application

    1. Create a new GitHub application with the following values:
    Field Value
    Application name any
    Homepage URL any
    Application description any
    Authorization callback URL https://injectify.samdd.me

    Setting up the server

    1. Clone this repo
    2. Copy server.config.example.js to server.config.js

    3. Replace the GitHub client_id and client_secret values with your newly created applications ones.

    4. Replace the mongodb value with your newly created mLab URL, in the format mongodb://<dbuser>:<dbpassword>@XXXX.mlab.com:XXXX/injectify.

    5. Run the following in a terminal at the root of the injectify repo:

    sudo npm i -g yarn
yarn global add typescript
yarn global add pm2
yarn run install:all
yarn run deploy

    Injectify will now be running over at http://localhost:3000
